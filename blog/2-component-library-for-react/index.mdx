# Introduction

In this post, I will walk you through how to build a component library for React using Vite and Tailwind. We will also see how to publish it to npm and use it in a React application. We will also integrate auto for versioning and release notes.

## Configure Auto

```bash
npm install -D auto @auto-it/all-contributors @auto-it/conventional-commits @auto-it/npm @auto-it/omit-commits @auto-it/omit-release-notes @auto-it/released
```

**Add labels**

Auto depends on labels which are added to PR to make a decision to release. So, we need to create labels on our repo.

Run `npx auto create-labels`

## Canary Release

**Make a canary release**

- Create a new branch from main
- Run `npx auto canary --force` and it should be successful
- At this point you should find a version in npm similar to `0.1.0--canary.457943b.0 -D`

## Release Action

Let's add a release action to Github which can handle releasing.

Before you push the following file, make sure you have added a secret `NPM_TOKEN` in your repo settings. This token would be used to publish to npm.

## Test Release

Note: You should already be in a branch at this point

- Raise a PR with "release" label, and merge it. 
- You should see a new release on Github with release notes (Check Actions tab for running)

At this point we have a npm package which is configured with vite to build, and auto to version and ship.
